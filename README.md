# Лабораторна 2

## Звіт до роботи

### Завдання: 
реалізувати клас `RegexFSM`, що приймає regex-вираз на вхід, компілює його у скінченний автомат і дозволяє перевіряти рядки на відповідність.

### Підтримуваний синтаксис regex:

- літери a-z, A-Z
- цифри 0-9
- `.` будь-який символ
- `*` нуль або більше повторень
- `+` одне або більше повторень

### Реалізовано:

- клас `TerminationState`
- метод `check_self` у `DotState`
- `__init__` і `check_self` у `AsciiState`
- `__init__` у `StarState`
- `__init__` і `check_self` у `PlusState`
- обробку операторів `*` і `+` у `RegexFSM.__init_next_state`
- метод `RegexFSM.check_string`

### Файли в репозиторії:

- `regex.py` основна реалізація
- `test_regex.py` додаткові тести з різними regex-патернами

## Коротке пояснення імплементації

Кожен токен regex отримує свій клас стану. Звичайні стани (буква, цифра, крапка) додаються по черзі у `StartState.next_states`.

Коли цикл побудови натрапляє на `*` або `+`, заготовка пропонує створити обгортку `StarState` чи `PlusState` навколо попереднього стану. У моїй реалізації обгортка зберігає обгорнутий стан як перший елемент свого `next_states` (бо успадкований `check_self` ітерує саме по `next_states`). Додатково я видаляю обгорнутий стан із `StartState.next_states`, бо тепер він представлений обгорткою, і двічі його там тримати немає сенсу.

Приклад побудови для regex `a*4`:

1. Додаю `AsciiState(a)`. Стан списку: `[a]`.
2. Зустрічаю `*`, створюю `StarState(a)`, видаляю `a` зі списку, додаю обгортку. Стан: `[StarState(a)]`.
3. Додаю `AsciiState(4)`. Стан: `[StarState(a), AsciiState(4)]`.

`check_string` обходить ланцюжок станів зі `StartState.next_states` рекурсивно. Для звичайного стану з'їдає один символ і йде далі. Для `*` і `+` використовую backtracking: пробую різну кількість повторень, починаючи з мінімально допустимої (0 для `*`, 1 для `+`), і на кожному варіанті продовжую матчинг решти regex. Якщо хоч одна кількість дала успішний шлях до кінця, повертаю True.

## Інструкції до запуску

Потрібен Python 3.10+

```
python regex.py
python test_regex.py
```

## Приклади запусків

`python regex.py` для regex `a*4.+hi`:

```
True
True
False
```

`python test_regex.py`:

```
'a*4.+hi':
  'aaaaaa4uhi' -> True  [OK]
        '4uhi' -> True  [OK]
        'meow' -> False  [OK]

'a+b*':
            '' -> False  [OK]
           'a' -> True  [OK]
       'aaabb' -> True  [OK]
           'b' -> False  [OK]

'H.+':
           'H' -> False  [OK]
          'Hi' -> True  [OK]
       'Hello' -> True  [OK]
       'hello' -> False  [OK]
```

## Висновок

Вході роботи - усі частини даного за умовою regex.py заповнив. Тести проходять: `python regex.py` дає очікувані `True, True, False`, `python test_regex.py` проходить на всіх кейсах. Для виконання роботи розібрав, як з патерну State зібрати скінченний автомат для regex і як працює backtracking для квантифікаторів `*` і `+`.
